# Project Reflection

## Overview

StackSave was designed as an AI spend optimization platform focused on helping organizations better understand and reduce recurring AI tooling costs.

The project evolved from a simple audit calculator into a full-stack SaaS-style platform with reporting, analytics, recommendations, persistence, and export capabilities.

---

# Key Learnings

## Full-Stack SaaS Architecture

This project significantly improved understanding of:
- frontend/backend separation
- scalable component structure
- state management
- persistence layers
- dynamic routing
- API-driven workflows

---

## Product-Oriented Engineering

A major learning outcome was thinking beyond pure coding and focusing on:
- user experience
- product flow
- executive reporting
- shareability
- presentation quality

The project was approached as a real SaaS MVP rather than only a technical assignment.

---

## Recommendation Systems

Building the audit engine introduced practical challenges around:
- optimization logic
- business heuristics
- overlap detection
- pricing analysis
- actionable recommendation generation

---

## Data Visualization

Implementing charts and executive dashboards improved understanding of:
- dashboard UX
- analytics presentation
- executive-level reporting
- data storytelling

---

# Challenges Faced

## AI API Limitations

During development, API quota and rate-limit issues were encountered while integrating external LLM providers.

This led to implementing fallback summary generation logic to maintain product reliability.

---

## PDF Export Compatibility

Modern Tailwind CSS color functions caused compatibility issues with PDF rendering libraries.

The solution involved switching rendering approaches to improve export reliability.

---

## Dynamic Report Architecture

Generating public shareable reports required:
- dynamic route generation
- UUID-based report architecture
- backend persistence coordination

---

# What Went Well

- Modular architecture
- Strong UI consistency
- Scalable audit engine structure
- Smooth end-to-end workflow
- Successful report export pipeline

# Final Thoughts

This project became significantly more advanced than initially planned and ultimately evolved into a strong portfolio-quality SaaS MVP.

The biggest takeaway was learning how product thinking, engineering, and user experience combine to create a compelling software platform.
